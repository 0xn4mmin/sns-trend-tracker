# TrendPulse — SNS 트렌드 인텔리전스 대시보드

유튜브 · 쇼츠 · 릴스 · X(트위터) · 스레드 · 틱톡 인기 콘텐츠와 AI 영상 모델·뉴스를
**한 화면에서, API 키 없이** 모아보는 웹 서비스입니다.
파이썬 표준 라이브러리만 사용하며 외부 패키지 설치가 전혀 필요 없습니다.

## 로컬 실행

```bash
python3 server.py
# → http://localhost:8778
```

## 웹 배포

서버가 `HOST`(기본 `0.0.0.0`)와 `PORT` 환경변수를 읽으므로 대부분의 PaaS에 그대로 올라갑니다.
Dockerfile과 Procfile이 모두 포함되어 있습니다.

### Render 원클릭 배포 (권장)

1. [render.com](https://render.com) 접속 → GitHub 계정으로 로그인
2. **New +** → **Blueprint** → 이 저장소(`sns-trend-tracker`) 선택
3. `render.yaml`이 자동 인식됨 → **Apply** 클릭
4. 1~2분 뒤 `https://trendpulse-xxxx.onrender.com` 형태의 공개 URL 발급

> 무료 플랜은 15분간 접속이 없으면 잠들었다가 첫 요청 때 ~50초 걸려 깨어납니다.
> 상시 구동이 필요하면 Starter 플랜($7/월)으로 올리면 됩니다.

### 기타 플랫폼

| 플랫폼 | 방법 |
|---|---|
| **Railway** | 저장소 연결만 하면 Procfile을 자동 인식 |
| **Fly.io** | `fly launch` → Dockerfile 자동 감지 |
| **Cloudtype** (국내) | Python 앱으로 배포, 시작 명령 `python server.py` |
| **VPS/EC2** | `nohup python3 server.py &` + nginx 리버스 프록시 |

> ⚠️ **Vercel/Netlify 등 서버리스는 부적합**합니다. 이 앱은 장시간 실행되는 서버가
> 외부 API를 프록시하고 메모리에 캐시하는 구조입니다.

### 환경변수

| 변수 | 기본값 | 설명 |
|---|---|---|
| `PORT` | `8778` | 서비스 포트 (PaaS가 자동 주입) |
| `HOST` | `0.0.0.0` | 바인드 주소 |
| `CACHE_TTL` | `3600` | 데이터 캐시 시간(초) |
| `DATA_DIR` | 앱 폴더 | 구독 계정·세션 JSON 저장 위치. 배포 시 영속 볼륨 경로로 지정하면 재시작 후에도 유지됩니다 |
| `TIKTOK_REGION` | `KR` | 틱톡 트렌딩 피드 지역 |
| `IG_COOKIE` | — | (선택) Instagram 로그인 세션 쿠키 → 릴스 인증 수집 |
| `X_COOKIE` | — | (선택) X 로그인 세션 쿠키(`auth_token`+`ct0`) → X 인증 수집 |
| `THREADS_COOKIE` | — | (선택) Threads 쿠키. 비우면 `IG_COOKIE` 공유 |
| `YT_COOKIE` | — | (선택) YouTube 쿠키 → 좋아요·연령제한 보강 |
| `ADMIN_TOKEN` | — | 설정 시, 앱 내 세션 설정 UI의 저장이 이 토큰(`X-Admin-Token` 헤더)을 요구 |

## 로그인 세션 기반 수집 (선택)

인스타그램·X·스레드는 무인증 조회를 자주 차단합니다. **본인 브라우저의 세션 쿠키**를 넣으면
인증 요청으로 안정적으로 수집합니다. 비밀번호나 자동 로그인은 사용하지 않습니다.

- **넣는 방법 2가지**: ① 앱 우상단 🔑 아이콘 → 플랫폼별 쿠키 붙여넣기, 또는 ② 위 환경변수
- **쿠키 얻기**: 브라우저에서 해당 플랫폼 로그인 → F12 → Network → 요청의 `cookie` 헤더 값 복사
- **안전 설계**: 각 쿠키는 **그 플랫폼 자신의 도메인으로만** 전송되고, 저장 값은 화면에 다시 표시되지 않습니다. 틱톡은 제3자 프록시(tikwm)를 경유하므로 쿠키를 받지 않습니다(유출 방지). 세션 쿠키는 비밀번호에 준하니 본인 계정·신뢰 기기에서만 사용하고 각 플랫폼 약관을 확인하세요.
- **공개 배포 시**: 세션 쿠키는 `sessions.json`(gitignore됨)이 아니라 **환경변수**로 넣고, UI 저장을 막으려면 `ADMIN_TOKEN`을 설정하세요.
- X는 GraphQL 쿼리 ID가 바뀌면 인증 수집이 멈추는데, 이때 `server.py`의 `X_QID_*`를 최신 값으로 갱신하면 됩니다(자동으로 무인증 방식으로 폴백하므로 서비스는 멈추지 않습니다).

## 기능

- **유튜브 / 쇼츠**: 카테고리(전체·AI·먹방·뷰티·브이로그·예능·영화/드라마·테크·지식·여행·동물) × 기간(오늘/이번 주/이번 달) 필터, 조회수/좋아요순 정렬(좋아요는 영상별 서버 보강), 카드 클릭 시 팝업 재생, 키워드 검색
- **AI 인사이트**: 새로 나온·트렌딩 영상 생성 모델(Hugging Face 실시간) + 국내외 AI 영상 뉴스(구글 뉴스 RSS)
- **릴스**: 인스타그램 웹 내부 API(무인증)로 구독 계정 최신 릴스 수집, 조회수/좋아요/댓글순 정렬
- **X**: 임베드용 공개 API(무인증)로 구독 계정 최신 포스트를 참여 지표와 함께 수집, 4가지 정렬
- **스레드**: 무인증 실시간 조회 시도, 차단 시 계정 바로가기 폴백
- **틱톡**: tikwm 공개 API로 실시간 인기 피드(한국) + 구독 계정 영상, 3가지 정렬
- **공통**: 구독 계정 추가/삭제(플랫폼별 저장), 1시간 서버 캐시 + 수동 새로고침, 다크/라이트 테마, 반응형(모바일 대응), 스켈레톤 로딩·토스트 알림

## 아키텍처

```
브라우저 (index.html — 정적 SPA)
   │  /api/videos · /api/reels · /api/x · /api/threads · /api/tiktok · /api/ai · /api/img
   ▼
server.py (파이썬 표준 라이브러리 HTTP 서버, 스레드 풀 병렬 수집)
   │  1시간 TTL 캐시 + 원본 장애 시 만료 캐시 폴백(stale-on-error)
   ▼
YouTube InnerTube · Instagram web API · X syndication · Threads GraphQL · tikwm · Hugging Face · Google News RSS
```

- 유튜브 데이터는 내부 검색 API(InnerTube)를 조회수순 정렬로 호출합니다. 공개 인기 급상승 피드는 2025년 폐지되어 검색 기반으로 구성했습니다.
- 인스타 릴스는 instagram.com 웹이 쓰는 내부 API(`web_profile_info` + 공개 앱 ID 헤더)를 무인증으로 호출합니다. 계정당 최근 12개 한도, 요청이 잦으면 일시 제한될 수 있습니다(캐시로 완화).
- 인스타/틱톡/트위터 CDN 썸네일은 핫링크 차단을 피해 서버 이미지 프록시(`/api/img`, 허용 호스트 화이트리스트 + 메모리 캐시)로 전달합니다.
- X는 임베드 위젯용 syndication API를 무인증으로 호출합니다. 조회수(impressions)가 없는 글은 조회수순 정렬에서 제외됩니다.
- 스레드는 GraphQL `doc_id`가 수시로 바뀌어 안정적 조회가 어렵습니다. `server.py`의 `THREADS_DOC_IDS`에 유효한 doc_id를 넣으면 즉시 실시간 조회가 동작합니다.
- 틱톡은 자체 API의 서명 검사를 우회하기 위해 tikwm 무료 공개 API를 사용합니다(낮은 동시성 + 1시간 캐시로 레이트리밋 완화).

## 상용화 시 유의사항

- 유튜브 InnerTube·인스타그램·X syndication 등 **비공식 API**를 사용하므로, 트래픽이 커지면 공식 API(YouTube Data API v3 등) 전환 또는 데이터 수집 파이프라인 분리를 권장합니다.
- 계정 목록이 전역 공유됩니다(모든 방문자가 같은 목록을 보고 수정 가능). 다중 사용자 서비스로 키우려면 사용자별 계정/인증 계층이 필요합니다.
- 각 플랫폼의 서비스 약관(ToS)과 저작권·데이터 사용 정책을 검토하세요.
